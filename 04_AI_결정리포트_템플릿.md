### 조합 규칙

```python
def generate_hybrid_report(scores, location, user):
    """점수와 문장을 조합하는 리포트 생성"""

    report = {}

    # 규칙 1: 점수 차이가 클 때 → 점수 강조 + 짧은 문장
    if scores['total'] >= 80 or scores['total'] <= 40:
        report['style'] = 'score_dominant'
        report['summary_length'] = 'short'  # 2~3문장

    # 규칙 2: 점수가 중간일 때 → 문장 강조 (뉘앙스 필요)
    elif 50 <= scores['total'] <= 70:
        report['style'] = 'text_dominant'
        report['summary_length'] = 'long'   # 4~6문장

    # 규칙 3: 축별 점수 편차가 클 때 → 강약점 문장 강조
    score_std = std_dev(scores['breakdown'].values())
    if score_std > 20:
        report['highlight_contrast'] = True
        # "접근성(91점)은 최상위지만, 비용(48점)은 주의가 필요합니다"

    # 규칙 4: 사용자 유형에 따른 어조 조정
    if user.risk_tolerance == 'low':
        report['tone'] = 'cautious'
        # "~이 우려되나", "신중한 검토가 필요합니다"
    elif user.risk_tolerance == 'high':
        report['tone'] = 'optimistic'
        # "충분히 도전할 만합니다", "기회가 큽니다"

    # 규칙 5: 비교 모드에서는 점수 테이블 + 결론 문장
    if report_type == 'comparison':
        report['format'] = 'table_first'
        # 표로 점수 비교 → 마지막에 추천 문장 1개

    return report
```

### 문장 톤 가이드라인

```
✅ 사용할 표현:
- "~이 예상됩니다" (객관적 전망)
- "~을 고려할 필요가 있습니다" (리스크 완곡)
- "~이 유리합니다/불리합니다" (명확한 판단)
- "데이터에 따르면 ~" (근거 제시)

❌ 피할 표현:
- "반드시 ~하세요" (명령형)
- "절대 ~하지 마세요" (과도한 단정)
- "확실히 성공합니다" (보장 불가)
- 감정적·주관적 표현
```
